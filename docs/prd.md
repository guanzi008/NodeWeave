# 企业级异地组网系统 PRD

Version: 1.1  
Date: 2026-03-10  
Owner: System Architecture / Product / Dev Team

## 1. 文档目的

本文档定义企业级异地组网系统的产品目标、范围、核心需求、版本边界与交付优先级，作为研发、测试、运维与项目管理的统一输入。

## 2. 项目目标

建设一套支持多平台、多版本、多网络形态的企业级异地组网系统，实现以下目标：

- 跨地域安全组网
- 终端与站点之间的 P2P 优先直连
- 无法打洞时自动回退 Relay
- 多客户端平台统一接入
- 与现有 VPN 或专线网络互通
- 企业级身份、ACL、审计与策略控制
- 内部 DNS、静态路由、出口节点、DNS 接管
- 应用使用统计与网络流量审计
- USB / 串口等工业设备远程接入

## 3. 目标用户

### 3.1 企业 IT 与安全团队

关注点：

- 异地办公接入

- 安全审计
- 用户与设备权限管理

### 3.2 工业与物联网运维团队

关注点：

- CLI 自动化管理

## 4. 核心场景

### 场景 A：员工远程接入企业内网

需求：

- 客户端安装后自动注册
- 用户登录并获得策略
- 仅允许访问授权子网和服务
- DNS 自动指向企业内部域名解析

### 场景 B：多个分支机构通过网关组网

需求：

- OpenWrt / Linux 网关加入 Overlay 网络
- 网关发布本地子网路由
- 总部访问分支网段无需每台终端安装客户端

### 场景 C：移动终端安全访问

需求：

- Android / iOS 通过系统 VPN 能力接入
- 支持出口节点
- 支持内部域名解析
- 支持基础审计与策略下发

### 场景 D：工业串口设备远程维护

需求：

- 将本地串口或 USB 设备映射到远端
- 支持虚拟串口或 TCP 封装
- 支持权限控制与操作审计

### 场景 E：混合网络互联

需求：

- 与现有 WireGuard、ZeroTier、EasyTier、Tailscale 网络互通
- 支持通过桥接网关导入外部路由和 DNS 记录
- 不要求协议级统一控制面

## 8. 关键功能需求

### 8.1 组网与连接

- 支持节点注册、离线重连和配置热更新
- 支持 NAT 打洞优先，Relay 自动回退
- 支持节点间点对点通信
- 支持站点级子网路由

### 8.2 出口节点

- 支持为用户、设备或组指定默认出口节点
- 支持是否允许访问本地 LAN
- 支持出口节点的 DNS 同步
- 支持按策略切换出口节点

### 8.3 静态路由

- 支持路由发布、撤销、冲突检测
- 支持优先级和下一跳控制
- 支持总部到分支、分支到分支的路由互通

### 8.4 DNS

- 支持 DNS 接管
- 支持内部权威域名解析
- 支持节点自动注册 A/AAAA 记录
- 支持服务发现记录与自定义记录
- 支持 UDP、TCP、DoH、DoT 接入

### 8.5 策略与 ACL

- 支持用户、设备、节点、子网、应用多维度策略
- 支持 allow / deny 显式规则
- 支持默认拒绝模式
- 支持策略审计与命中追踪

### 8.6 审计与统计

- 记录流量五元组、流量大小、连接持续时间
- 记录策略命中结果
- 记录应用使用时长、启动次数、会话次数
- 支持用户维度、设备维度、节点维度聚合

### 8.7 USB / 串口转发

- 支持 USB/IP 模式
- 支持串口映射为虚拟 COM / tty
- 支持 TCP 封装串口数据
- 支持设备授权、独占锁和会话审计

## 9. 非功能需求

### 9.1 安全

- 所有控制面通信必须双向认证
- 数据面必须端到端加密
- 密钥轮换可配置
- 审计数据必须防篡改和可追溯

### 9.2 性能

- 节点加入网络到获取配置应小于 5 秒
- P2P 建链成功后端到端额外延迟尽量控制在 10ms 到 30ms 级别
- Relay 回退场景应支持区域级扩容
- 控制面应支持 10 万级节点元数据管理

### 9.3 可用性

- 控制面支持多实例部署
- Relay 支持多地域部署
- 客户端支持断线重连
- 配置下发失败可自动回滚至上次稳定版本

### 9.4 可运维性

- 所有关键模块输出结构化日志
- 提供 Prometheus 指标
- 具备审计检索能力
- 支持灰度升级

## 10. 兼容策略定义

兼容不等于完全协议互通。产品中“兼容 WireGuard / Tailscale / ZeroTier / EasyTier”定义如下：

- 可通过桥接网关接入既有网络
- 可导入或发布静态路由
- 可同步或代理 DNS 解析
- 可映射 ACL 到本系统策略模型
- 可通过出口节点或边界网关实现互访

不承诺：

- 直接复用第三方控制面
- 节点身份跨产品自动互认
- 无适配器情况下的协议级隧道互通

## 11. 成功指标

### 11.1 技术指标

- P2P 建链成功率大于 70%，其余场景自动 Relay
- 控制服务器配置下发成功率大于 99.9%
- 客户端异常退出率低于既定桌面与移动端阈值
- 核心审计数据落库成功率大于 99.99%

### 11.2 产品指标

- 首批部署可支撑 3 类典型场景：远程办公、站点互联、工业串口维护
- 安装到上线平均时长控制在 30 分钟以内
- 管理员可以在单控制台完成节点、用户、策略、DNS、路由管理

## 13. 风险与约束

- iOS 与 macOS 的系统扩展能力受 Apple 平台约束，功能需按 Network Extension 可用边界设计。
- OpenWrt 与嵌入式资源受限，不能直接复用桌面全功能实现。
- 应用使用统计在不同平台的系统开放程度不同，需要分平台定义精度和授权方式。
- WireGuard、Tailscale、ZeroTier、EasyTier 之间不存在统一控制协议，桥接和路由互通必须通过网关适配实现。
